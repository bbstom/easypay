# 钱包配置页面重构方案

## 目标

将 `WalletConfigPage.jsx` 重构为更清晰的结构：

### 1. 主标签保持不变
- 状态监控 (status)
- 基础配置 (basic) - **需要添加子标签**
- 能量租赁 (energy) - **从"资源配置"改名**

### 2. 基础配置添加 3 个子标签
- **监控API配置** (api) - 默认子标签
  - TronGrid 节点配置
  - ZAN 节点配置
  - 多节点策略说明
  
- **钱包地址** (wallet)
  - 当前钱包地址显示
  - 私钥输入和验证
  - 测试连接功能
  - 安全提示
  
- **自动转账** (transfer)
  - 启用/禁用自动转账
  - 最大重试次数
  - TRX 最低余额预警
  - USDT 最低余额预警

### 3. 资源配置改名为能量租赁
- 标签名称: "能量租赁" (energy)
- 内容保持不变

## 实施步骤

### 步骤 1: 添加子标签状态管理
```javascript
const currentSubTab = searchParams.get('subtab') || 'api';
```

### 步骤 2: 在基础配置标签中添加子标签导航
```jsx
{currentTab === 'basic' && (
  <>
    {/* 子标签导航 */}
    <div className="bg-white rounded-xl border border-slate-200 mb-6 overflow-hidden">
      <div className="flex border-b border-slate-200">
        <button onClick={() => setSearchParams({ tab: 'basic', subtab: 'api' })}>
          监控API配置
        </button>
        <button onClick={() => setSearchParams({ tab: 'basic', subtab: 'wallet' })}>
          钱包地址
        </button>
        <button onClick={() => setSearchParams({ tab: 'basic', subtab: 'transfer' })}>
          自动转账
        </button>
      </div>
    </div>
    
    {/* 子标签内容 */}
    {currentSubTab === 'api' && <APIConfig />}
    {currentSubTab === 'wallet' && <WalletConfig />}
    {currentSubTab === 'transfer' && <TransferConfig />}
  </>
)}
```

### 步骤 3: 拆分原有的基础配置内容

#### 监控API配置 (api)
- 从原来的"基础配置"中提取 API 节点配置部分
- 包含：
  - TronGrid 配置
  - ZAN 配置
  - 多节点策略说明
  - 保存按钮

#### 钱包地址 (wallet)
- 从原来的"基础配置"中提取钱包相关部分
- 包含：
  - 当前钱包地址显示
  - 私钥输入框
  - 验证私钥按钮
  - 测试连接按钮
  - 测试结果显示
  - 安全提示

#### 自动转账 (transfer)
- 从原来的"转账配置"部分移动过来
- 包含：
  - 启用自动转账开关
  - 最大重试次数
  - TRX 最低余额预警
  - USDT 最低余额预警
  - 保存按钮

### 步骤 4: 修改主标签导航
将"资源配置"改为"能量租赁"

3. **第 642-1063 行**: 将"资源配置"改为"能量租赁"
4. **AdminLayout 中的标签导航**: 修改标签名称

## 优势

1. **更清晰的结构**: 每个配置项都有独立的页面
2. **更好的用户体验**: 用户可以专注于一个配置项
3. **更易维护**: 代码结构更清晰，便于后续维护
4. **符合多钱包系统**: 为后续添加多钱包管理做准备
